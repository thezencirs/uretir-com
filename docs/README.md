## Documentation rules

- Documentation describes reality first, then clearly labels intended future state.
- A behavior change that affects users, data, architecture, performance, SEO, accessibility, security, or operations must update the relevant document in the same change set.
- A decision that is expensive to reverse needs a dated entry in [decision-log.md](./decision-log.md).
- Documents use concise English for cross-functional and international readability. Product-facing copy may remain in Turkish.
- Do not add a document merely to satisfy a checklist. Each page must answer a decision-making question for its reader.
